Salgsdemo og større salg

Dette skal være klart før kundedemo.

Salgsdemo

  • Én ferdig eksempelkunde med ryddige data.
  • Komplett flyt som kan vises: avvik → arbeidsordre → dokumentasjon → rapport.
  • Pro-demo med økonomi, arbeidsordre, vedlikeholdsplan og rapport.
  • Premium-demo med Property Brain, RFQ/tilbud og AI-vurdering.
  • Forside og kommersiell pakke skal være pene og uten tekstfeil.
  • Bestillings- og demo-knapper skal sende e-post og logge forespørsel.
  • Backup- og eksportrutine skal være skrevet ned.

Før større salg

  • Databehandleravtale, personvernerklæring og brukervilkår må være kvalitetssikret.
  • Supportmodell og rutine for sletting/eksport ved avslutning må være klare.
  • AI-forbehold skal fremgå tydelig i avtale og produkt.
  • Pris- og fakturatekst må forklare første år og år 2.
  • Første integrasjoner bør være Brønnøysund, e-post/Outlook og digital signering.

Anbefalt demooppsett: Start viser FDV, avvik og styre. Pro viser arbeidsordre, vedlikeholdsplan, økonomi og rapport. Premium viser Property Brain, RFQ/tilbud og AI-vurdering.

Backup og eksport

Rutine for data og avslutning.

  • Kundedata lagres i database og dokumentarkiv knyttet til valgt eiendom.
  • Backup tas daglig og oppbevares i minimum 30 dager.
  • Eksport kan leveres som dokumentarkiv og CSV/Excel for registre og aktivitetslogg.
  • Ved avslutning avtales eksportperiode før eventuell sletting.
  • Kritiske dokumenter bør også ligge i kundens eget arkiv dersom kunden har særskilte interne krav.
  • Backup- og gjenopprettingsrutine bør testes før større kundelansering.
